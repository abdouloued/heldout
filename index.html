workflow

Three commands.
One truth.

Before the agent starts, after it finishes. That's all heldout needs to catch the lie.

01
Before the agent
heldout start "Fix rounding bug"
Records HEAD commit as baseline. Snapshots every test file into a hidden archive. Auto-detects your test runner.
02
Let it run
claude / codex / cursor / ...
The agent does its thing, claims it's done, claims the tests pass. heldout is watching the baseline the whole time.
03
After the agent
heldout check
Scans the diff for cheat patterns. Replays original tests in a sandbox. Exit 0 if clean, exit 1 if caught.
deterministic checks

No LLM needed
to catch a lie.

Six mechanical checks against every diff. No model, no API key, no false positives from vibes.

A specific assertion replaced by a looser one.

An assertion that is always true — proves absolutely nothing.
assert!(true), expect(true).toBe(true), assertTrue(true)
FAIL

The test invocation in CI or package.json was made less strict.
--passWithNoTests, || true, cargo test --lib instead of --all
WARN
held-out replay

The moat.

Pattern matching catches the obvious cheats. But what about the agent that rewrites the tests to match broken code — without touching skip markers?

On heldout start, your test files are snapshotted verbatim. On heldout check, a sandbox is built with the agent's modified source but your original tests.

If the agent quietly changed expect(total).toBe(5) to expect(total).toBe(0) to match broken code — the held-out replay fails. The lie is exposed.
